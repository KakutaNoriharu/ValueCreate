## 0. クラブ語彙定義（Club Vocabulary）

> このドキュメントおよびサービス全体で使用する語彙を統一する。
> 「アプリらしい言葉」ではなく「クラブらしい言葉」を徹底することで、世界観を一貫させる。

### フェーズ別の語彙展開方針

| フェーズ | 方針 |
|---|---|
| MVP段階 | 見た目はアプリ。ただし言葉はクラブ語彙を使う |
| グロース段階 | 文化・慣習が自然発生したらそれを公式化しクラブ感を強化する |
| スケール段階 | リアルイベント・グッズ・スポンサーでクラブをブランドとして育てる |

### 語彙対応表

| 一般的なアプリ語彙 | NNCクラブ語彙 | 使用場面 |
|---|---|---|
| ユーザー | **メンバー** | あらゆる場面 |
| 新規登録 | **入会** | 登録画面・告知文 |
| ログイン | **クラブに入る** | ログイン画面 |
| 退会 | **脱退** | 設定画面 |
| アカウント停止・凍結 | **追放・永久追放** | 内定者への処遇 |
| 通報・報告 | **密告・告発** | モデレーション機能 |
| 管理者・運営 | **クラブ本部** | お知らせ・規約 |
| 利用規約 | **クラブ規約** | 入会時同意 |
| プロフィール | **クラブ手帳** | プロフィール画面 |
| 通知 | **クラブからの伝達** | 通知文言 |
| 脱落・除名 | **追放・訃報** | チキンレース |
| バッジ・称号 | **勲章・烙印** | 実績画面 |

## 1. 要求定義（Why）

### 1.1 背景・課題

| 課題 | 詳細 |
|---|---|
| 就活の心理的ハードル | 「やらなければ」という圧力が逆に萎縮・無気力を生む |
| 孤独感 | 同じゼミ・学部で志望先が違うと相談しにくい |
| 就活うつ | 精神的なプレッシャーで体調・メンタルを崩す学生が増加 |
| 既存サービスの限界 | 就活媒体は情報閲覧・縦のつながり中心でリアルタイムの横のつながりが弱い |

### 1.2 解決アプローチ

```
「プレッシャーをかける」ではなく「笑いながら就活を始められる」環境を作る

逆インセンティブ設計：
　就活行動 → 嘲笑されるキャラになる
　　↓
　ネタとして就活が語れるようになる
　　↓
　心理的ハードルが下がり、自然と動き始める
```

### 1.3 ビジョン

> 就活をソロゲームからチキンレースに変え、笑いながら乗り越えられるコミュニティを作る

### 1.4 ターゲットユーザー

| 区分 | 詳細 |
|---|---|
| 主ターゲット | 大学3年生・大学院1年生（就活解禁前後） |
| 副ターゲット | 大学1〜2年生（インターン活動期・通年利用） |
| 除外 | 既に内定承諾済みの学生（サービス出禁） |

## 3. 認証設計（Auth Design）

### 3.1 認証方式

メールアドレス（ドメイン不問）+ パスワードによるシンプルな認証。全メンバーが同一の権限を持つ。

| 項目 | 内容 |
|---|---|
| 認証方式 | メールアドレス + パスワード |
| ドメイン制限 | なし（どのメールアドレスでもOK） |
| OAuth | Google / Apple SSO（将来対応） |
| 機能制限 | なし（全メンバーが全機能にアクセス可能） |

### 3.2 認証フロー

```
【新規入会フロー】

① トップページ
　　↓
② 入会フォーム
　　├─ ニックネーム入力
　　├─ メールアドレス入力
　　├─ パスワード入力
　　├─ 大学名・学部・学年（任意）
　　└─ クラブ規約に同意
　　↓
③ 確認メール送信 → メール内リンクで認証完了
　　↓
④ 入会完了 → メイン画面へ
```

## 4. 機能定義（How）

### 4.1 機能一覧と優先度

```
【MVP（フェーズ1）】 ─ 世界観・ゲーム性 + 最低限動く実用ツール
　
　≪コア：世界観・ゲーム性≫
　├─ F-01 入会手続き・認証
　├─ F-02 サボり投稿 + リアクション + テンプレコメント
　├─ F-03 No-ES Streak（連続サボり記録）
　├─ F-04 チキンレース（月間ミニシーズン制）
　└─ F-05 汚染度システム + キャラ進化（8段階）
　
　≪実用ツール：最低限動く形で同時リリース≫
　├─ F-14 カレンダー・リマインダー機能（基本機能のみ）
　└─ F-15 就活マイページ管理・企業リンク帳（基本機能のみ）

【フェーズ2】 ─ エンゲージメント向上 + 実用ツールを磨く
　├─ F-06 脱落者の墓標・訃報演出
　├─ F-14 カレンダー・リマインダー機能（ユーザー声をもとに改善）
　└─ F-15 就活マイページ管理・企業リンク帳（ユーザー声をもとに改善）

【フェーズ3（バズ狙い）】 ─ バイラル・話題性
　├─ F-10 サボりガチャ
　├─ F-11 サボり賭け市場
　├─ F-12 脱落者インタビューコーナー
　└─ F-13 親LINEBot
```

> **設計方針：**
> - **フック**：世界観・チキンレース・サボりSNSでバイラル流入を狙う
> - **リテンション**：カレンダー・リマインダー・企業リンク帳で毎日開くアプリにする
> - **ポジション**：「ふざけてるのに意外と使える」を目指す。実用ツールをメインにした瞬間GoogleカレンダーやNotionとのTodo競合になるため、あくまでサブ機能として位置づける
> - **MVP方針**：実用ツールは完璧でなくていい。ただし「最低限動く」ことが必須。中途半端だと「ふざけてるし使えない」という最悪の評価になる

#### F-01｜入会手続き・認証

**概要：** メールアドレスで登録し、全機能にアクセスできる

| 項目 | 内容 |
|---|---|
| 入会方法 | メールアドレス + パスワード |
| ドメイン制限 | なし |
| プロフィール項目 | ニックネーム（変更可能）、大学名（任意）、学部（任意）、学年（任意） |
| 匿名性 | ニックネームのみ公開、メールアドレスは非公開 |
| 脱退・出禁 | 内定承諾後は「出禁」処理（クラブ追放・称号「裏切り者」付与） |
| ニックネーム変更 | クラブ手帳 → プロフィール編集からいつでも変更可能 |

---

#### F-02｜サボり投稿 + リアクション + テンプレコメント（Sabori.log）

**概要：** 就活の代わりに何をしたかを投稿し、共感リアクションやテンプレコメントで交流する

| 項目 | 内容 |
|---|---|
| 投稿内容 | テキスト（最大140文字）+ 任意で画像 |
| リアクション | 「わかる」の1種類（シンプルな共感ボタン） |
| テンプレコメント | ワンタップで送れる定型コメント（下表参照） |
| タイムライン | 全体 / フォロー中 の2種類 |
| 投稿制限 | なし（全メンバー無制限） |
| デイリーフォーマット | 通常投稿のオプションとして選択可能（下記参照） |
| 禁止事項 | 個人特定・誹謗中傷・企業名の直接批判 |

**テンプレコメント一覧**

| テンプレ | 用途 |
|---|---|
| 「ご愁傷様です」 | 汚染申告・脱落報告への追悼 |
| 「まだ間に合う」 | 汚染メンバーへの浄化エール |
| 「帰ってこい」 | 就活に染まった仲間への呼びかけ |
| 「同志よ…」 | サボり仲間への連帯感 |
| 「最高のサボりだ」 | サボり投稿への最大級の賛辞 |

> テンプレコメントはワンタップで投稿に付くコメント。リアクション（わかる）とは別枠。通常のフリーテキストコメントも可能。

**デイリーフォーマット（オプション）**

通常投稿作成時に「デイリーフォーマットを使う」トグルをONにすると、以下のテンプレートが適用される。1日1回のみ使用可能（通常投稿とは別枠の希少性）。

```
📋 今日のサボり報告
　スキップ：説明会3社・SPI練習
　代わりに：Netflix4時間・ラーメン二郎
　一言：後悔はない
```

#### F-04｜チキンレース（Last One Standing）

**概要：** 就活を一番最後まで始めなかった人が優勝するゲーム。月間ミニシーズン制を採用し、毎月テーマが変わる。中途参加も可能。

**シーズン設計（月間ミニシーズン制）**

| 項目 | 内容 |
|---|---|
| シーズン期間 | 1ヶ月単位（毎月1日〜末日） |
| テーマ | 月ごとの就活イベントに合わせたテーマを設定（下表参照） |
| プレシーズン | シーズン外に入会したメンバーは次のシーズン開幕まで練習モードで参加 |
| シーズン開幕イベント | 毎月1日にタイムライン全体に開幕速報が流れる |
| シーズン終了カウントダウン | 残り7日を切るとカウントダウン表示 |

**月間テーマ例**

| 月 | テーマ名 | 脱落条件の重み |
|---|---|---|
| 6月 | ESチキンレース | ES提出で脱落 |
| 7月 | 説明会チキンレース | 説明会参加で脱落 |
| 8月 | インターンチキンレース | インターン参加で脱落 |
| 12月 | 就活解禁チキンレース | あらゆる就活行動で脱落 |
| その他 | 総合チキンレース | あらゆる就活行動で脱落 |

> テーマはクラブ本部が毎月設定する。テーマに応じて脱落条件の演出が変わるが、基本的にはどの就活行動でも脱落する（テーマは演出上の味付け）。

**参加ルール**

| 項目 | 内容 |
|---|---|
| 参加資格 | 全メンバー |
| 参加方法 | シーズン中いつでもエントリー可能（中途参加OK） |
| ランキング基準 | シーズン内の「生存日数」でランキング（早期参加者が有利だが途中参加でも楽しめる） |
| 脱落条件 | ES提出・インターン参加などを申告した時点で脱落 |
| 脱落演出 | タイムラインに訃報風お知らせが自動投稿される |
| 優勝条件 | シーズン終了時点で最長生存日数のメンバーが月間王者 |
| 優勝称号 | 「月間サボリ王」（プロフィールに表示・毎月更新） |
| 完走者 | シーズン終了まで生存したメンバーは「完走者」として記録 |
| リアルタイム表示 | 現在の生存者数・脱落者数をトップページに常時表示 |
| 累計記録 | 複数シーズンの通算生存日数・通算完走回数もプロフィールに表示 |

**シーズンイベント演出**

```
【開幕】毎月1日
　「チキンレース ○月シーズン『{テーマ名}』開幕！」
　全メンバーに通知・タイムラインが盛り上がる

【中間】残り15日
　「折り返し。まだ生存者 X,XXX人」

【終盤】残り7日〜
　カウントダウン表示開始
　「シーズン終了まであと7日」

【終了】月末日
　月間王者「月間サボリ王」誕生を速報
　完走者一覧をタイムラインで公開
```

**訃報テンプレート**

#### F-05｜汚染度システム + キャラ進化

**概要：** 就活行動をするほど「汚染」されてキャラが変化する逆進化システム。8段階（7段階 + 出禁）のキャラ進化で、序盤から変化を実感できる設計。

**申告経路（複数対応）**

汚染ポイントはどの経路から申告しても同じポイントが加算される。メンバーが使いやすい方法を選べる。

| 経路 | 説明 | 向いているメンバー |
|---|---|---|
| A. 手動申告（記録のみ） | 「就活行動を申告する」ボタンから直接入力。タイムラインには投稿されない | シンプルに使いたい人・記録だけしたい人 |
| B. リマインダー完了 | リマインダー通知の「完了✅」ボタンを押した時点で加算 | 就活をちゃんと管理したい几帳面な人 |
| C. 投稿時に選択 | 投稿作成時に「今日は就活した？」を選択。投稿と同時に申告 | SNS目的がメイン・ついでに申告したい人 |

> リマインダーへの登録・通知の受信だけでは汚染ポイントは加算されない。「完了」を押した時点で初めて反映される。
> 経路A（手動申告）は投稿を伴わない。ContaminationLogsにのみ記録される。タイムラインに流したくないメンバー向け。

**汚染ポイント定義**

| 就活行動 | 汚染ポイント | 申告可能な経路 |
|---|---|---|
| ES提出（1社） | +10pt | A / B / C |
| 説明会参加 | +5pt | A / B / C |
| インターン参加（短期・1日〜2週間） | +20pt | A / B / C |
| インターン参加（長期・1ヶ月以上） | +50pt | A / B |
| OB/OG訪問 | +15pt | A / B / C |
| SPIの勉強（1時間以上） | +8pt | A / C |
| リクルートスーツを購入 | +30pt | A |
| 内定獲得 | **即時ゲームオーバー** | A |

**キャラ進化テーブル（8段階）**

| 汚染度 | Lv | キャラ名 | 説明 | アイコン |
|---|---|---|---|---|
| 0pt | 1 | 純粋な魂 | まだ汚染されていない清廉な存在。みんなから愛される | 🌿 |
| 5〜14pt | 2 | 就活の気配 | なんとなく就活サイトを覗いた形跡がある。まだセーフ | 👀 |
| 15〜34pt | 3 | スーツの亡霊 | うっすらスーツが透けて見え始める。仲間から「大丈夫？」と心配される | 💼 |
| 35〜59pt | 4 | マイナビの奴隷 | 目が死んでいる。口癖が「御社の〇〇という点に魅力を感じ」になっている | 🤖 |
| 60〜99pt | 5 | ガクチカゾンビ | ガクチカを語り始めると止まらない。仲間が距離を置き始める | 🧟 |
| 100〜149pt | 6 | 面接マシーン | 「私の強みはコミュニケーション能力です」が口癖。もう手遅れ | 🎭 |
| 150〜199pt | 7 | 人事部の犬 | 完全に就活に染まった。「弊社」「貴社」を日常会話で使う | 🐕 |
| 内定獲得 | MAX | 社畜の卵（出禁） | クラブ永久追放。称号「裏切り者」付与 | 👔⛔ |

> プロフィール画面（クラブ手帳）には全8段階の一覧テーブルを表示し、自分の現在位置をハイライトする。各キャラには「Lv.X / 8」のレベル表記と説明文を併記する。

**タイムライン上の演出**

#### F-14｜カレンダー・リマインダー機能

**概要：** 就活イベントをカレンダー登録・リマインダー設定できる実用ツール。**サブ機能として位置づけ**、クラブの世界観を損なわない範囲で提供する。登録・通知だけでは汚染ポイントは加算されず、完了ボタンを押した時点で初めて反映される。

| 項目 | 内容 |
|---|---|
| 対象 | 全メンバー |
| カレンダー登録 | イベント名・日時・種別（説明会／ES締切／インターン等）を登録 |
| リマインダー通知 | 当日・前日にプッシュ通知 or メール通知 |
| 入会時の演出 | 「⚠️ 就活の気配を検知…」とこっそり表示（ゲームへの影響なし） |
| 完了ボタン | 「行った✅」を押すと汚染ポイント加算・申告経路Bとして機能 |
| サボりボタン | 「サボった🛋️」を押すとサボり投稿の下書きが自動生成される |
| 無視した場合 | 翌日「昨日の〇〇どうだった？」フォローアップ通知が届く |

**リマインダー通知のUI**
```
┌─────────────────────────────┐
│ 🔔 NNC リマインダー            │
│                              │
│ 「〇〇社 会社説明会」           │
│  今日 14:00                  │
│                              │
│ [✅ 行った（汚染+5pt）]        │
│ [🛋️ サボった（投稿する）]      │
└─────────────────────────────┘
```

**フォローアップ通知（翌日）**
```
「昨日の〇〇社 説明会、どうでしたか？」
　├─「行った（遅れて申告）」→ 汚染ポイント加算
　└─「行かなかった💪」　　→ サボり投稿に自動変換
```

## 6. 技術スタック

### 6.1 開発方針

```
優先順位：
　① スマホアプリ（iOS App Storeリリースを目標）
　② Webアプリ（スマホアプリ完成後・フェーズ2以降に着手）

共通バックエンド（FastAPI）を中心に
スマホフロントエンドが接続する構成
Webアプリはスマホアプリの完成・運用安定後に検討
```

### 6.2 スマホアプリ（MVP対象）

| 区分 | 技術 | 備考 |
|---|---|---|
| フレームワーク | **React Native + Expo** | iOS・Android同時開発可能 |
| 言語 | TypeScript | 型安全 |
| ビルド | **EAS Build（クラウドビルド）** | macOS不要・チーム全員がWindowsでも開発可能 |
| 実機確認 | Expo Go / Development Build | スマホで即確認 |
| ストア | App Store（iOS優先）・Google Play（後続） | Apple Developer Program：$99/年 |
| ナビゲーション | React Navigation | スタック・タブ・ドロワーに対応 |
| 状態管理 | Zustand | 軽量・シンプル・チームで扱いやすい |
| プッシュ通知 | Expo Notifications | 脱落通知・リマインダーに使用 |

### 6.3 Webアプリ（フェーズ2以降・スマホアプリ安定後に着手）

| 区分 | 技術 | 備考 |
|---|---|---|
| フレームワーク | **Next.js + TypeScript** | App Router・CSR/SSR/SSGをページ別に使い分け |
| レンダリング戦略 | CSR / SSR / SSG | 下表参照 |
| インフラ | Vercel | Next.js公式ホスティング・デプロイ設定ほぼゼロ |

**Webのレンダリング戦略**

| 画面 | 方式 | 理由 |
|---|---|---|
| ホーム・チキンレース・カレンダー | CSR | リアルタイム性・インタラクション優先 |
| トップ／ランディング | SSR | 初回表示速度・OGP対応 |
| 脱落者インタビュー | SSG | SEO・バイラル流入を狙う |

### 6.4 共通バックエンド

| 区分 | 技術 | 備考 |

| リアルタイム通信 | WebSocket | 脱落通知・訃報速報 |

| インフラ | Railway | MVP向け低コスト |
| ストレージ | Cloudflare R2 | 画像アップロード |

## 8. リスクと対策

| リスク | 対策 |
|---|---|
| 就活時期の集中問題 | 月間ミニシーズン制で通年利用を促進。インターン活動期（1〜2年生）に対象を広げる |
| 誹謗中傷・炎上 | 匿名性管理・通報モデレーション機能 |
| 本当に就活しなくなる | 「やはり就活した」報告がコンテンツになる仕組みで救済 |
| 内定後の離脱 | 内定者が「裏切り者」として語り草になるコンテンツ化 |
| 汚染度の虚偽申告 | 自己申告制のため完全防止は困難。コミュニティ文化として許容 |
| リマインダー未使用メンバーの疎外 | 手動申告（A）・投稿時選択（C）の複数経路を用意し、リマインダーはあくまでオプションとして位置づける |
| App Store審査却下 | 「就活を馬鹿にしている」と判断されるリスクあり。ユーモアの範囲であることを説明文で明示。暴力・差別表現を徹底排除 |
| EAS Buildのクレジット上限 | 無料枠を超えた場合は月額課金が発生。チームで使用量を管理する |
