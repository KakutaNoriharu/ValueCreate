# Figma の使い方ガイド（アプリアイコン・キャラ素材制作向け）

## 1. 始め方

### アカウント作成・料金
- https://www.figma.com でGoogleアカウントまたはメールで登録
- **無料プラン（Starter）で十分**。個人利用ならファイル3つまで、ページ数無制限
- ブラウザ版とデスクトップアプリ版がある。どちらでもOKだが、デスクトップ版の方がショートカットが安定する

### 初期画面
- ログイン後「Drafts」に入る → 右上の「+ Design file」で新規ファイル作成
- 1ファイル = 1プロジェクト（「NNC素材」みたいな名前にしておく）

---

## 2. 基本操作

### キャンバス操作
| 操作 | Mac | Windows |
|---|---|---|
| 拡大・縮小 | Cmd + スクロール | Ctrl + スクロール |
| キャンバス移動 | Space + ドラッグ | Space + ドラッグ |
| 全体を表示 | Cmd + Shift + 1 | Ctrl + Shift + 1 |
| 選択範囲にズーム | Cmd + 2 | Ctrl + 2 |

### ツールバー（左上）
| ショートカット | ツール | 用途 |
|---|---|---|
| V | 選択ツール | オブジェクト選択・移動 |
| F | フレーム | アートボード作成（iPhoneサイズ等） |
| R | 長方形 | 四角を描く |
| O | 楕円 | 丸を描く |
| L | 線 | 直線を引く |
| P | ペン | パスで自由に描く |
| T | テキスト | 文字入力 |

### 右サイドバー（プロパティ）
- オブジェクトを選択すると右側に表示される
- **Design タブ**：位置(X,Y)、サイズ(W,H)、色（Fill）、線（Stroke）、角丸、不透明度
- **Prototype タブ**：画面遷移の設定（今回は使わない）

## 3. フレームとグループ

### フレーム（F）
- Figmaにおける「アートボード」。書き出し単位になる
- 使い方：Fキー → キャンバスでドラッグ、または右サイドバーからプリセット選択
- アイコン制作なら「40×40」「120×120」のカスタムサイズでフレームを作る

```
フレーム作成の例（キャラアイコン用）：
  F キー → キャンバスでドラッグ → 右サイドバーで W:120, H:120 に設定
  → フレーム名を「pure_large」などに変更（左サイドバーのレイヤー名をダブルクリック）
```

### グループ（Cmd/Ctrl + G）
- 複数オブジェクトをまとめる。フレームと違い、はみ出した部分もそのまま表示される
- パーツが多いキャラをまとめる時に使う

### フレーム vs グループの使い分け
- **フレーム**：書き出し単位にしたいもの（アイコン1個 = 1フレーム）
- **グループ**：フレーム内でパーツをまとめたいだけの時

## 4. 色と塗り

### 色の設定
1. オブジェクトを選択
2. 右サイドバーの「Fill」の色をクリック
3. カラーピッカーが開く → HEX値を直接入力できる

### NNCカラーパレット（直接入力用）
```
Primary（黒）    : 1a1a1a
OnPrimary（白）  : f5f5f5
汚染（赤）       : e24b4a
生存（緑）       : 1d9e75
説明会（青）     : 378add
インターン       : ef9f27
グレー           : 888780
純粋な魂（薄緑） : e1f5ee
汚染済（薄赤）   : faece7
```

### 背景透過
- フレームを選択 → 右サイドバーの「Fill」のチェックを外す → 背景なし（透過）
- PNG書き出し時に透過が保持される

## 5. 図形でキャラを作る（ベクター編集）

### 基本図形の組み合わせ
シンプルなキャラは基本図形の組み合わせで作れる：

```
例：「純粋な魂」キャラ
1. O キー → 正円を描く（顔）→ サイズ 80×80、Fill: e1f5ee
2. O キー → 小さい楕円を2つ（目）→ サイズ 8×10、Fill: 1a1a1a
3. P キー → 口を曲線で描く
4. 全選択 → Cmd+G でグループ化
5. フレーム（120×120）の中央に配置
```

### ブーリアン演算（図形の結合・切り抜き）
複数の図形を選択して、上部ツールバーのブーリアンメニューから：
| 操作 | 効果 | 使い所 |
|---|---|---|
| Union | 結合（合体） | 複数パーツを1つのシルエットにする |
| Subtract | 前面で背面を切り抜き | 穴を開ける・くり抜く |
| Intersect | 重なった部分だけ残す | マスク的な使い方 |
| Exclude | 重なった部分を除外 | ドーナツ型を作る |

### ペンツール（P）で自由に描く
1. P キーでペンツール起動
2. クリックで直線のアンカーポイントを打つ
3. クリック + ドラッグで曲線のハンドルを出す
4. 閉じるときは最初のポイントをクリック
5. Escで確定

コツ：アンカーポイントは少なめに。後からダブルクリックでパスを編集できる。

## 6. コンポーネント化

### コンポーネントとは
- 再利用可能なパーツ（UIコンポーネントと同じ概念）
- 元を変更するとコピー先（インスタンス）にも反映される

### 作り方
1. オブジェクトやフレームを選択
2. 上部ツールバーの ◇ アイコン（Create component）をクリック、または Cmd/Ctrl + Alt + K
3. レイヤー名が紫の ◇ アイコンになる = コンポーネント化完了

### 使い方
- 左サイドバーの「Assets」タブからドラッグ＆ドロップで配置
- 配置されたものは「インスタンス」（白抜きの ◇ アイコン）

### NNCでの活用例
```
コンポーネント設計：
├── CharacterIcon（キャラアイコン・Variantsで8段階切替）
├── TabIcon（タブアイコン・active/inactive の2状態）
└── NNCLogo（ロゴ）
```

---

## 7. Variants（バリアント）

キャラアイコンの8段階を1つのコンポーネントで管理できる機能。

### 作り方
1. 8つのキャラアイコンフレーム（pure, whisper, ghost, ...）をそれぞれコンポーネント化
2. 全部選択 → 右クリック → 「Combine as variants」
3. プロパティ名を「stage」に変更
4. 各バリアントの値を「pure」「whisper」「ghost」...に設定

### 使い方
- インスタンスを配置 → 右サイドバーで stage を切り替えるとキャラが変わる
- デザイン確認時に全段階をすぐ見比べられて便利

## 8. 書き出し（Export）

### 個別書き出し
1. 書き出したいフレーム or オブジェクトを選択
2. 右サイドバー最下部の「Export」セクション
3. 「+」をクリックして書き出し設定を追加

### 書き出し設定
| 設定 | 値 | 用途 |
|---|---|---|
| Format | PNG | キャラアイコン・タブアイコン（背景透過対応） |
| Format | SVG | ベクターアイコン（拡大縮小に強い） |
| Scale | 1x | 通常サイズ |
| Scale | 2x | Retina対応（React Native向けに推奨） |
| Scale | 3x | 高解像度ディスプレイ用 |

### React Native 向けの書き出しベストプラクティス

```
書き出す際の命名規則（ファイル名）：
  pure.png      → @1x
  pure@2x.png   → @2x（React Nativeが自動で選択してくれる）
  pure@3x.png   → @3x

Expo の場合は @2x, @3x のサフィックスを付けて同じフォルダに置くだけで
デバイスに応じて自動選択される。
```

### 一括書き出し
1. 書き出したい全フレームを選択（Cmd/Ctrl + クリックで複数選択）
2. それぞれに Export 設定を追加
3. 右サイドバーの「Export (N layers)」ボタンで一括書き出し

- ページごとに用途を分けると管理しやすい

4. 完成したら PNG @1x/@2x/@3x で書き出し
5. `apps/mobile/assets/characters/` に配置
