# NNC アプリ 動作確認手順

動作確認はバックエンドとモバイルの2つを起動するだけでOKです。
ローカルは SQLite を使用するため、PostgreSQL や Redis の準備は不要です。

## 1. バックエンド（FastAPI）を起動

```bash
cd /home/knori/ValueCreate/backend
python3 -m venv venv && source venv/bin/activate   # 初回のみ
pip install -r requirements.txt                     # 初回のみ
cp .env.example .env                                # 初回のみ（SQLiteがデフォルト）
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

起動したら以下で動作確認できます：

- API ドキュメント（Swagger UI）: **http://localhost:8000/docs** ← ここで全APIを画面から叩けます
- DB は自動で `nnc.db`（SQLite）が作られるので、DBサーバーの準備は不要です

## 2. モバイル（Expo）を起動

```bash
cd /home/knori/ValueCreate/apps/mobile
npm install          # 初回のみ
npx expo start
```

接続先は `services/api.ts` で `EXPO_PUBLIC_API_URL`（未設定なら `http://localhost:8000`）を見ています。
実機（iPhone等）で確認する場合は LAN IP 指定が必要です：

```bash
# start-dev.sh の IP を自分のPCのLAN IPに書き換えてから
EXPO_PUBLIC_API_URL=http://192.168.11.19:8000 npm run start:lan
```

確認方法は3通り：

- **実機**: Expo Go アプリで QR コードを読む（`start:lan` で起動）
- **Web**: `npm run web` → ブラウザで開く（一番手軽）
- **エミュレータ**: `npm run android` / `npm run ios`

## テスト用ユーザーでログインして確認する（推奨）

認証済みのテストユーザーとサンプルデータ（投稿・チキンレース・カレンダー・企業リンク帳）を
使って、ログインから各画面までを一通り確認する手順です。

> **最重要:** ログインにはバックエンド（FastAPI）が起動している必要があります。
> ログインボタンを押すとアプリは `http://localhost:8000/api/auth/login` にアクセスします。
> バックエンドが起動していないと「ログインに失敗しました」になります。
> 起動用ターミナルは確認が終わるまで**閉じずに開いたまま**にしてください。

ターミナルは2つ使います。

- **ターミナルA**: バックエンド（uvicorn）— 起動しっぱなしにする
- **ターミナルB**: モバイル（Expo）— 起動しっぱなしにする

### 手順1: テストデータを投入する（ターミナルA）

サンプルデータを一括投入するシードスクリプトを実行します（冪等。何度実行しても重複しません）。
初回や DB を作り直したときに実行すればOKです。

```bash
cd /home/knori/ValueCreate/backend
source venv/bin/activate
python -m scripts.seed_test_user
```

成功すると `✅ シード完了` とログイン情報が表示されます。

| 項目 | 値 |
| --- | --- |
| メールアドレス | `test@nnc.dev` |
| パスワード | `password123` |

このユーザーは `email_verified=True`（認証済み）なので、メール認証を経由せずにログインできます。

### 手順2: バックエンドを起動する（ターミナルA・起動しっぱなし）

シードと同じターミナルでそのまま起動します。

```bash
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

ブラウザで `http://localhost:8000/docs` が開けば起動成功です。
**このターミナルは閉じない**でください（ログイン中ずっと必要）。

### 手順3: 認証バイパスを OFF にする

ログイン画面で認証したいので、バイパスを無効化します。
`apps/mobile/.env` を次のように設定（または該当行を削除）：

```bash
EXPO_PUBLIC_DEV_BYPASS_AUTH=0
```

### 手順4: モバイルを起動する（ターミナルB・起動しっぱなし）

環境変数を反映させるため、必ずキャッシュをクリアして起動します。

```bash
cd /home/knori/ValueCreate/apps/mobile
npx expo start -c
```

Web で手軽に見る場合は起動後に `w` キー（または `npm run web`）。

### 手順5: アプリからログインする

1. トップ画面（⚰️内定ないクラブへようこそ）で **「すでにメンバーの方はこちら →」** をタップ
2. ログイン画面で上記のメール / パスワードを入力
   - メールアドレス: `test@nnc.dev`
   - パスワード: `password123`
3. **「クラブに入る」** ボタンをタップ
4. ログインに成功するとメインタブ（下部タブ）に切り替わります。
   実トークンが発行され、以降の API はデータ込みで動作します。

> ボタンを押して「ログインに失敗しました」と出る場合は、ほぼ手順2のバックエンドが
> 起動していない（またはターミナルを閉じてしまった）ことが原因です。
> ターミナルAで uvicorn が動いているか、`http://localhost:8000/docs` が開けるか確認してください。

### 手順6: ログイン後、各タブでデータを確認する

ログイン後は下部タブで各画面を確認できます。シードで投入したデータが表示されればOKです。

| 下部タブ | 操作・確認できるもの |
| --- | --- |
| **ホーム** | 投稿3件（サボり日報・通常・訃報）がタイムラインに並ぶ。リアクションを押して反応するか確認 |
| **チキンレース** | 「第1期 内定回避シーズン」に生存中（生存37日）と表示される |
| **投稿（＋）** | 記録＆投稿モーダルが開く。投稿するとホームのタイムライン先頭に追加される |
| **イベント** | 上部タブで「カレンダー」「企業リンク帳」を切り替え |
| └ カレンダー | ES締め切り・合同説明会の2件が表示される |
| └ 企業リンク帳 | 株式会社ニートワークスが表示される |
| **プロフィール** | ニックネーム「テスト無い男」／汚染度120／streak 37日。右上などから設定画面へ遷移できる |

### 手順7: ログアウトして別アカウントを試す（任意）

プロフィール → 設定画面からログアウトすると、再びトップ画面（AuthStack）に戻ります。
別ユーザーで入会フローを試したり、再ログインを確認できます。

## 認証ゲートを開発用にバイパスする（任意）

トップ画面で「入会」または「ログイン」をしないとメイン画面（ホーム・タイムライン等）に進めません。
UI だけ素早く確認したい場合は、開発用バイパスで認証ゲートを素通りできます。

`apps/mobile/.env` に以下を設定します（リポジトリには既に追加済み）：

```bash
EXPO_PUBLIC_DEV_BYPASS_AUTH=1
```

環境変数は起動時に読み込まれるため、**キャッシュをクリアして再起動**してください：

```bash
cd /home/knori/ValueCreate/apps/mobile
npx expo start -c
```

これでトップ画面をスキップして、いきなりメインタブが開きます。
元に戻すには `.env` の値を `0` にする（または行を削除する）→ `-c` 付きで再起動。

**注意:**
- このバイパスは `__DEV__`（開発ビルド）限定で、本番ビルドでは無効です。
- バイパス中はログイントークンが無いため、認証必須 API は 401 になります。
  画面レイアウトは見えますが、データは空 / エラー表示になる箇所があります（落ちはしません）。

## 注意点

1. `start-dev.sh` の IP `192.168.11.19` は、あなたの PC の現在の LAN IP と一致している必要があります（実機接続時）。
2. メール認証（Resend）は `.env` がプレースホルダーのままだと送信されません。
   ローカル確認では認証コードをログ / DB から確認する形になる可能性があります。
