## 0. クラブ語彙定義（Club Vocabulary）

> このドキュメントおよびサービス全体で使用する語彙を統一する。
> 「アプリらしい言葉」ではなく「クラブらしい言葉」を徹底することで、世界観を一貫させる。

### フェーズ別の語彙展開方針

| フェーズ | 方針 |
|---|---|
| MVP段階 | 見た目はアプリ。ただし言葉はクラブ語彙を使う |
| グロース段階 | 文化・慣習が自然発生したらそれを公式化しクラブ感を強化する |
| スケール段階 | リアルイベント・グッズ・スポンサーでクラブをブランドとして育てる |

### 語彙対応表

| 一般的なアプリ語彙 | NNCクラブ語彙 | 使用場面 |
|---|---|---|
| ユーザー | **メンバー** | あらゆる場面 |
| 新規登録 | **入会** | 登録画面・告知文 |
| ログイン | **クラブに入る** | ログイン画面 |
| 退会 | **脱退** | 設定画面 |
| アカウント停止・凍結 | **追放・永久追放** | 内定者への処遇 |
| 通報・報告 | **密告・告発** | 就活警察機能 |
| 管理者・運営 | **クラブ本部** | お知らせ・規約 |
| 利用規約 | **クラブ規約** | 入会時同意 |
| プロフィール | **クラブ手帳** | プロフィール画面 |
| 通知 | **クラブからの伝達** | 通知文言 |
| ランキング | **クラブ名鑑** | ランキング画面 |
| 脱落・除名 | **追放・訃報** | チキンレース |
| バッジ・称号 | **勲章・烙印** | 実績画面 |

## 1. 要求定義（Why）

### 1.1 背景・課題

| 課題 | 詳細 |
|---|---|
| 就活の心理的ハードル | 「やらなければ」という圧力が逆に萎縮・無気力を生む |
| 孤独感 | 同じゼミ・学部で志望先が違うと相談しにくい |
| 就活うつ | 精神的なプレッシャーで体調・メンタルを崩す学生が増加 |
| 既存サービスの限界 | 就活媒体は情報閲覧・縦のつながり中心でリアルタイムの横のつながりが弱い |

### 1.2 解決アプローチ

```
「プレッシャーをかける」ではなく「笑いながら就活を始められる」環境を作る

逆インセンティブ設計：
　就活行動 → 嘲笑されるキャラになる
　　↓
　ネタとして就活が語れるようになる
　　↓
　心理的ハードルが下がり、自然と動き始める
```

### 1.3 ビジョン

> 就活をソロゲームからチキンレースに変え、笑いながら乗り越えられるコミュニティを作る

### 1.4 ターゲットユーザー

| 区分 | 詳細 |
|---|---|
| 主ターゲット | 大学3年生・大学院1年生（就活解禁前後） |
| 副ターゲット | 大学1〜2年生（インターン活動期・通年利用） |
| 除外 | 既に内定承諾済みの学生（サービス出禁） |

## 3. 認証設計（Auth Design）

### 3.1 認証ステータス定義

| ステータス | 称号 | 取得方法 | 扱い |
|---|---|---|---|
| `university` | 🎓 **正規就活生** | 大学メール（.ac.jp）で認証 | フル機能アクセス・ランキング集計対象 |
| `normal` | 🕵️ **素性不明の怪しいやつ** | 普通メール or Google/Apple SSO | 一部機能制限・ランキング集計対象外 |

> 「怪しいやつ」ラベルはネガティブではなく、NNCのユーモアとして演出する。

---

### 3.2 認証ステータス別の機能差分

| 機能 | 🎓 正規就活生（university） | 🕵️ 怪しいやつ（normal） |
|---|---|---|
| チキンレース参加 | ✅ フル参加・優勝資格あり | ❌ 観戦のみ |
| サボり投稿 | ✅ 無制限 | ⚠️ 1日3回まで |
| 汚染度・キャラ進化 | ✅ フル機能 | ❌ 表示なし |
| 大学別ランキング集計 | ✅ 集計対象 | ❌ 集計対象外 |
| 認証済み🎓バッジ | ✅ 表示される | ❌ なし |
| タイムライン閲覧 | ✅ 全体閲覧 | ✅ 全体閲覧（間口は広く） |
| いいね・リアクション | ✅ 全種類（わかる／尊い／草） | ⚠️ 「草」のみ |
| No-ES Streak | ✅ フル機能・称号あり | ⚠️ カウントのみ・称号なし |
| デイリー投稿 | ✅ 参加可能 | ❌ 参加不可 |
| 就活警察・密告 | ✅ 送受信可能 | ⚠️ 受信のみ |

---

### 3.3 認証フロー

```
【新規入会フロー】

① トップページ
　　↓
② 登録方法を選択
　　├─ A. 大学メールで登録 → .ac.jpドメイン検証 → 確認メール送信 → 認証完了
　　│　　　→ ステータス：university（🎓正規就活生）
　　│
　　└─ B. 普通メール or Google/Apple SSOで登録 → そのまま入会完了
　　　　　→ ステータス：normal（🕵️怪しいやつ）

【アップグレードフロー】

normalメンバーが設定画面から大学メールを追加認証
　　↓
.ac.jpドメイン検証 → 確認メール送信 → 認証完了
　　↓
ステータスが university に昇格
　　↓
タイムラインに「@○○の正体が判明しました」演出が流れる 🎉
```

### 3.4 「怪しいやつ」のまま居続けることへの演出

- 長期間未認証のままだと「謎のベテラン」称号がつく（180日以上）
- 認証しないことも一つのキャラクター戦略として成立
- NNCの逆張り文化と相性がよく、あえて認証しない層が生まれる

## 4. 機能定義（How）

### 4.1 機能一覧と優先度

```
【MVP（フェーズ1）】 ─ 世界観・ゲーム性 + 最低限動く実用ツール
　
　≪コア：世界観・ゲーム性≫
　├─ F-01 入会手続き・認証（二層化）
　├─ F-02 サボり投稿 + いいね
　├─ F-03 No-ES Streak（連続サボり記録）
　├─ F-04 チキンレース（脱落管理）
　└─ F-05 汚染度システム + キャラ進化
　
　≪実用ツール：最低限動く形で同時リリース≫
　├─ F-14 カレンダー・リマインダー機能（基本機能のみ）
　└─ F-15 就活マイページ管理・企業リンク帳（基本機能のみ）

【フェーズ2】 ─ エンゲージメント向上 + 実用ツールを磨く
　├─ F-06 脱落者の墓標・訃報演出
　├─ F-07 全国・大学別サボりランキング
　├─ F-08 就活警察・密告機能
　├─ F-09 デイリー投稿フォーマット
　├─ F-14 カレンダー・リマインダー機能（ユーザー声をもとに改善）
　└─ F-15 就活マイページ管理・企業リンク帳（ユーザー声をもとに改善）

【フェーズ3（バズ狙い）】 ─ バイラル・話題性
　├─ F-10 サボりガチャ
　├─ F-11 サボり賭け市場
　├─ F-12 脱落者インタビューコーナー
　└─ F-13 親LINEBot
```

> **設計方針：**
> - **フック**：世界観・チキンレース・サボりSNSでバイラル流入を狙う
> - **リテンション**：カレンダー・リマインダー・企業リンク帳で毎日開くアプリにする
> - **ポジション**：「ふざけてるのに意外と使える」を目指す。実用ツールをメインにした瞬間GoogleカレンダーやNotionとのTodo競合になるため、あくまでサブ機能として位置づける
> - **MVP方針**：実用ツールは完璧でなくていい。ただし「最低限動く」ことが必須。中途半端だと「ふざけてるし使えない」という最悪の評価になる

### 4.2 機能詳細

#### F-01｜入会手続き・認証（二層化）

**概要：** 大学メールまたは普通メールで登録し、認証ステータスに応じて機能を分ける

| 項目 | 内容 |
|---|---|
| 入会方法A | 大学メール（.ac.jp）認証 → 🎓正規就活生ステータス |
| 入会方法B | 普通メール or Google/Apple SSO → 🕵️怪しいやつステータス |
| プロフィール項目 | ニックネーム、大学名（任意）、学部（任意）、学年（任意） |
| 匿名性 | ニックネームのみ公開、メールアドレスは非公開 |
| アップグレード | 後から大学メールを追加認証してステータス昇格が可能 |
| 脱退・出禁 | 内定承諾後は「出禁」処理（クラブ追放・称号「裏切り者」付与） |

---

#### F-02｜サボり投稿 + いいね（Sabori.log）

**概要：** 就活の代わりに何をしたかを投稿し、共感いいねを競う

| 項目 | 内容 |
|---|---|
| 投稿内容 | テキスト（最大140文字）+ 任意で画像 |
| いいね種類 | 「わかる」「尊い」「草」の3種類のリアクション |
| タイムライン | 全体 / フォロー中 / 大学内 の3種類 |
| 投稿制限 | 🎓：無制限　🕵️：1日3回まで |
| 禁止事項 | 個人特定・誹謗中傷・企業名の直接批判 |

---

#### F-03｜No-ES Streak（連続サボり記録）

**概要：** 何もしない日が続くほどストリークが伸びる、Duolingoの逆

| 項目 | 内容 |
|---|---|
| カウント方法 | 毎日0時にチェックイン（就活行動なし）で+1日 |
| リセット条件 | 汚染ポイント加算行動を申告した時点でリセット |
| 対象 | 🎓：フル機能・称号あり　🕵️：カウントのみ・称号なし |

**称号テーブル（🎓正規就活生のみ）**

| 連続日数 | 称号 | 演出 |
|---|---|---|
| 1〜2日 | 様子見中 | — |
| 3〜6日 | 準備運動中 | — |
| 7日 | 1週間の猛者 | 🎉バッジ付与 |
| 30日 | 悟りの境地 | 🏅バッジ付与・ランキング掲載 |
| 60日 | 無敵モード | 🔥バッジ付与 |
| 100日 | 伝説のサボリスト | 👑称号・殿堂入り |

#### F-04｜チキンレース（Last One Standing）

**概要：** 就活を一番最後まで始めなかった人が優勝するゲーム（🎓正規就活生のみ参加可）。シーズン制を採用し、中途参加も可能。

**シーズン設計**

| 項目 | 内容 |
|---|---|
| シーズン期間 | 就活解禁日（12月1日）〜内定式（翌年10月1日）の約10ヶ月 |
| プレシーズン | シーズン外に入会したメンバーは次のシーズン開幕まで練習モードで参加 |
| シーズン開幕イベント | 12月1日にタイムライン全体に開幕速報が流れる |
| シーズン終了カウントダウン | 内定式まで残り30日を切るとカウントダウン表示 |

**参加ルール**

| 項目 | 内容 |
|---|---|
| 参加資格 | 🎓正規就活生のみ（🕵️怪しいやつは観戦のみ） |
| 参加方法 | シーズン中いつでもエントリー可能（中途参加OK） |
| ランキング基準 | シーズン内の「生存日数」でランキング（早期参加者が有利だが途中参加でも楽しめる） |
| 脱落条件 | ES提出・インターン参加などを申告した時点で脱落 |
| 脱落演出 | タイムラインに訃報風お知らせが自動投稿される |
| 優勝条件 | シーズン終了時点で最長生存日数のメンバーが優勝 |
| 優勝称号 | 「令和の無職王」（殿堂入り・プロフィールに永久表示） |
| 完走者 | シーズン終了まで生存したメンバーは「完走者」として記録 |
| リアルタイム表示 | 現在の生存者数・脱落者数をトップページに常時表示 |

**シーズンイベント演出**

```
【開幕】12月1日
　「チキンレース シーズン○ 開幕！」
　全メンバーに通知・タイムラインが盛り上がる

【中間】残り90日
　「あと90日。まだ生存者 X,XXX人」

【終盤】残り30日〜
　カウントダウン表示開始
　「内定式まであと30日」

【終了】10月1日
　最終生存者「令和の無職王」誕生を速報
　完走者一覧をタイムラインで公開
```

**訃報テンプレート**
```
━━━━━━━━━━━━━━━━━━
　　　　訃　報

　　@{ユーザー名} 氏は
　　本日、{脱落理由} されました。

　　在サボり日数：{日数}日
　　汚染度：{ポイント}pt

　　ご冥福をお祈りいたします。
━━━━━━━━━━━━━━━━━━
```

#### F-05｜汚染度システム + キャラ進化

**概要：** 就活行動をするほど「汚染」されてキャラが変化する逆進化システム（🎓正規就活生のみ）

**申告経路（複数対応）**

汚染ポイントはどの経路から申告しても同じポイントが加算される。メンバーが使いやすい方法を選べる。

| 経路 | 説明 | 向いているユーザー |
|---|---|---|
| A. 手動申告 | 「就活行動を申告する」ボタンから直接入力 | シンプルに使いたい人・カレンダー不要な人 |
| B. リマインダー完了 | リマインダー通知の「完了✅」ボタンを押した時点で加算 | 就活をちゃんと管理したい几帳面な人 |
| C. 投稿時に選択 | サボり投稿作成時に「今日は就活した？」を選択 | SNS目的がメイン・ついでに申告したい人 |

> リマインダーへの登録・通知の受信だけでは汚染ポイントは加算されない。「完了」を押した時点で初めて反映される。

**汚染ポイント定義**

| 就活行動 | 汚染ポイント | 申告可能な経路 |
|---|---|---|
| ES提出（1社） | +10pt | A / B / C |
| 説明会参加 | +5pt | A / B / C |
| インターン参加（短期・1日〜2週間） | +20pt | A / B / C |
| インターン参加（長期・1ヶ月以上） | +50pt | A / B |
| OB/OG訪問 | +15pt | A / B / C |
| SPIの勉強（1時間以上） | +8pt | A / C |
| リクルートスーツを購入 | +30pt | A |
| 内定獲得 | **即時ゲームオーバー** | A |

**キャラ進化テーブル**

| 汚染度 | キャラ名 | 説明 | アイコン |
|---|---|---|---|
| 0pt | 純粋な魂 | まだ汚染されていない清廉な存在。みんなから愛される | 🌿 |
| 10〜34pt | スーツの亡霊 | うっすらスーツが透けて見え始める。仲間から「大丈夫？」と心配される | 💼 |
| 35〜99pt | マイナビの奴隷 | 目が死んでいる。口癖が「御社の〇〇という点に魅力を感じ」になっている | 🤖 |
| 100〜199pt | ガクチカゾンビ | ガクチカを語り始めると止まらない。仲間が距離を置き始める | 🧟 |
| 内定獲得 | 社畜の卵（出禁） | クラブ永久追放。称号「裏切り者」付与 | 👔⛔ |

**タイムライン上の演出**
- 汚染度10pt以上：投稿に「⚠️汚染注意」バッジ表示
- 汚染度35pt以上：投稿に「🚨就活汚染注意🚨」バナー自動付与
- 仲間から「浄化スタンプ」を送れる（「まだ間に合う」「帰ってこい」）

#### F-06｜脱落者の墓標・訃報演出

| 項目 | 内容 |
|---|---|

#### F-14｜カレンダー・リマインダー機能

**概要：** 就活イベントをカレンダー登録・リマインダー設定できる実用ツール。**サブ機能として位置づけ**、クラブの世界観を損なわない範囲で提供する。登録・通知だけでは汚染ポイントは加算されず、完了ボタンを押した時点で初めて反映される。

| 項目 | 内容 |
|---|---|
| 対象 | 全メンバー（🎓／🕵️ 両方） |
| カレンダー登録 | イベント名・日時・種別（説明会／ES締切／インターン等）を登録 |
| リマインダー通知 | 当日・前日にプッシュ通知 or メール通知 |
| 入会時の演出 | 「⚠️ 就活の気配を検知…」とこっそり表示（ゲームへの影響なし） |
| 完了ボタン | 「行った✅」を押すと汚染ポイント加算・申告経路Bとして機能 |
| サボりボタン | 「サボった🛋️」を押すとサボり投稿の下書きが自動生成される |
| 無視した場合 | 翌日「昨日の〇〇どうだった？」フォローアップ通知が届く |

**リマインダー通知のUI**
```
┌─────────────────────────────┐
│ 🔔 NNC リマインダー            │
│                              │
│ 「〇〇社 会社説明会」           │
│  今日 14:00                  │
│                              │
│ [✅ 行った（汚染+5pt）]        │
│ [🛋️ サボった（投稿する）]      │
└─────────────────────────────┘
```

**フォローアップ通知（翌日）**
```
「昨日の〇〇社 説明会、どうでしたか？」
　├─「行った（遅れて申告）」→ 汚染ポイント加算
　└─「行かなかった💪」　　→ サボり投稿に自動変換
```

**概要：** 就活企業のマイページURLを一覧で管理できる実用ツール。**サブ機能として位置づけ**、パスワードは保存しない設計で安全性を担保する。

## 6. 技術スタック

### 6.1 開発方針

```
優先順位：
　① スマホアプリ（iOS App Storeリリースを目標）
　② Webアプリ（スマホアプリ完成後に着手）

共通バックエンド（FastAPI）を中心に
スマホ・Webの両フロントエンドが接続する構成
```

### 6.2 スマホアプリ（優先）

| 区分 | 技術 | 備考 |
|---|---|---|
| フレームワーク | **React Native + Expo** | iOS・Android同時開発可能 |
| 言語 | TypeScript | 型安全・Web側と共通化できる |
| ビルド | **EAS Build（クラウドビルド）** | macOS不要・チーム全員がWindowsでも開発可能 |
| 実機確認 | Expo Go | スマホ1台あれば即確認できる |
| ストア | App Store（iOS優先）・Google Play（後続） | Apple Developer Program：$99/年 |
| ナビゲーション | React Navigation | スタック・タブ・ドロワーに対応 |
| 状態管理 | Zustand | 軽量・シンプル・チームで扱いやすい |
| プッシュ通知 | Expo Notifications | 脱落通知・リマインダーに使用 |

### 6.3 Webアプリ（後続）

| 区分 | 技術 | 備考 |
|---|---|---|
| フレームワーク | **Next.js + TypeScript** | App Router・CSR/SSR/SSGをページ別に使い分け |
| レンダリング戦略 | CSR / SSR / SSG | 下表参照 |
| インフラ | Vercel | Next.js公式ホスティング・デプロイ設定ほぼゼロ |

## 8. リスクと対策

| リスク | 対策 |
|---|---|
| 就活時期の集中問題 | インターン活動期（1〜2年生）に対象を広げ通年化 |
| 誹謗中傷・炎上 | 匿名性管理・通報モデレーション機能 |
| 本当に就活しなくなる | 「やはり就活した」報告がコンテンツになる仕組みで救済 |
| 内定後の離脱 | 内定者が「裏切り者」として語り草になるコンテンツ化 |
| 汚染度の虚偽申告 | 自己申告制のため完全防止は困難。コミュニティ文化として許容 |
| リマインダー未使用メンバーの疎外 | 手動申告（A）・投稿時選択（C）の複数経路を用意し、リマインダーはあくまでオプションとして位置づける |
| 大学メール失効（卒業後） | contact_emailとuniversity_emailを分離管理し、失効しても継続利用可能 |
| 🕵️メンバーの荒らし | 通常投稿を1日3回に制限・リアクション制限で影響を最小化 |
| App Store審査却下 | 「就活を馬鹿にしている」と判断されるリスクあり。ユーモアの範囲であることを説明文で明示。暴力・差別表現を徹底排除 |
| EAS Buildのクレジット上限 | 無料枠を超えた場合は月額課金が発生。チームで使用量を管理する |
