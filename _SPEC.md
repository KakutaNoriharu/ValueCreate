# Naitei Nai Club (NNC) — 要求・要件・機能定義書

> *「内定ない」を誇れる唯一のSNS*

---

## 1. 要求定義（Why）

### 1.1 背景・課題

| 課題 | 詳細 |
|---|---|
| 就活の心理的ハードル | 「やらなければ」という圧力が逆に萎縮・無気力を生む |
| 孤独感 | 同じゼミ・学部で志望先が違うと相談しにくい |
| 就活うつ | 精神的なプレッシャーで体調・メンタルを崩す学生が増加 |
| 既存サービスの限界 | 就活媒体は情報閲覧・縦のつながり中心でリアルタイムの横のつながりが弱い |

### 1.2 解決アプローチ

```
「プレッシャーをかける」ではなく「笑いながら就活を始められる」環境を作る

逆インセンティブ設計：
　就活行動 → 嘲笑されるキャラになる
　　↓
　ネタとして就活が語れるようになる
　　↓
　心理的ハードルが下がり、自然と動き始める
```

### 1.3 ビジョン

> 就活をソロゲームからチキンレースに変え、笑いながら乗り越えられるコミュニティを作る

### 1.4 ターゲットユーザー

| 区分 | 詳細 |
|---|---|
| 主ターゲット | 大学3年生・大学院1年生（就活解禁前後） |
| 副ターゲット | 大学1〜2年生（インターン活動期・通年利用） |
| 除外 | 既に内定承諾済みの学生（サービス出禁） |

## 2. 要件定義（What）

### 2.1 ビジネス要件

| 要件ID | 内容 | 優先度 |
|---|---|---|
| BR-01 | 認証方式の二層化（大学メール認証 / 普通メール登録）による信頼度の可視化 | 高 |
| BR-02 | 就活時期（3〜6月）だけでなく通年利用できる設計 | 中 |
| BR-03 | SNSシェアによるバイラル拡散を前提とした設計 | 高 |
| BR-04 | 誹謗中傷・炎上リスクへの対策 | 高 |

### 2.2 ユーザー要件

| 要件ID | ユーザーストーリー | 優先度 |
|---|---|---|
| UR-01 | 就活生として、今日サボった内容を投稿して共感を得たい | 高 |
| UR-02 | 就活生として、自分の就活しなさっぷりを数値で確認したい | 高 |
| UR-03 | 就活生として、他の人も同じようにサボっていることを見て安心したい | 高 |
| UR-04 | 就活生として、チキンレースで最後まで残って称号を得たい | 高 |
| UR-05 | 就活生として、就活行動によってキャラが変化していく様子を楽しみたい | 中 |
| UR-06 | 就活生として、脱落者（ES提出者）の動向をリアルタイムで知りたい | 中 |
| UR-07 | 就活生として、大学別・学部別のサボりランキングを見たい | 低 |
| UR-08 | ユーザーとして、大学メール認証で「正規就活生」バッジを得て信頼性を示したい | 中 |
| UR-09 | ユーザーとして、普通メールでも気軽に登録してサービスを試したい | 高 |

### 2.3 システム要件

| 要件ID | 内容 | 優先度 |
|---|---|---|
| SR-01 | 認証方式の二層化（大学メール / 普通メール）と、それに応じた機能制限の実装 | 高 |
| SR-02 | 投稿・いいね・コメントのリアルタイム反映 | 高 |
| SR-03 | 脱落イベント発生時のリアルタイム通知（WebSocket） | 高 |
| SR-04 | 汚染ポイントの自動計算・キャラ状態の自動更新 | 高 |
| SR-05 | 連続サボり日数（Streak）の毎日自動カウント | 高 |
| SR-06 | 大学別・学部別の集計・ランキング生成（認証済みユーザーのみ集計対象） | 中 |
| SR-07 | 不適切コンテンツの通報・モデレーション機能 | 高 |
| SR-08 | スマートフォン対応（レスポンシブデザイン） | 高 |
| SR-09 | 普通メールユーザーが後から大学メールを追加認証できるアップグレード機能 | 中 |

### 2.4 非機能要件

| 区分 | 要件 |
|---|---|
| 可用性 | 99%以上の稼働率（就活解禁期は特に重要） |
| パフォーマンス | 投稿・通知のレスポンスは1秒以内 |
| セキュリティ | 個人情報の匿名化・大学メールと連絡用メールの分離管理 |
| スケーラビリティ | 就活解禁時期のアクセス集中に対応できる設計 |
| プライバシー | 汚染度・サボり記録は本人が公開範囲を選択できる |

## 3. 認証設計（Auth Design）

### 3.1 認証ステータス定義

| ステータス | 称号 | 取得方法 | 扱い |
|---|---|---|---|
| `university` | 🎓 **正規就活生** | 大学メール（.ac.jp）で認証 | フル機能アクセス・ランキング集計対象 |
| `normal` | 🕵️ **素性不明の怪しいやつ** | 普通メール or Google/Apple SSO | 一部機能制限・ランキング集計対象外 |

> 「怪しいやつ」ラベルはネガティブではなく、NNCのユーモアとして演出する。

---

### 3.2 認証ステータス別の機能差分

| 機能 | 🎓 正規就活生（university） | 🕵️ 怪しいやつ（normal） |
|---|---|---|
| チキンレース参加 | ✅ フル参加・優勝資格あり | ❌ 観戦のみ |
| サボり投稿 | ✅ 無制限 | ⚠️ 1日3回まで |
| 汚染度・キャラ進化 | ✅ フル機能 | ❌ 表示なし |
| 大学別ランキング集計 | ✅ 集計対象 | ❌ 集計対象外 |
| 認証済み🎓バッジ | ✅ 表示される | ❌ なし |
| タイムライン閲覧 | ✅ 全体閲覧 | ✅ 全体閲覧（間口は広く） |
| いいね・リアクション | ✅ 全種類（わかる／尊い／草） | ⚠️ 「草」のみ |
| No-ES Streak | ✅ フル機能・称号あり | ⚠️ カウントのみ・称号なし |
| デイリー投稿 | ✅ 参加可能 | ❌ 参加不可 |
| 就活警察・密告 | ✅ 送受信可能 | ⚠️ 受信のみ |

---

### 3.3 認証フロー

```
【新規登録フロー】

① トップページ
　　↓
② 登録方法を選択
　　├─ A. 大学メールで登録 → .ac.jpドメイン検証 → 確認メール送信 → 認証完了
　　│　　　→ ステータス：university（🎓正規就活生）
　　│
　　└─ B. 普通メール or Google/Apple SSOで登録 → そのまま登録完了
　　　　　→ ステータス：normal（🕵️怪しいやつ）

【アップグレードフロー】

normalユーザーが設定画面から大学メールを追加認証
　　↓
.ac.jpドメイン検証 → 確認メール送信 → 認証完了
　　↓
ステータスが university に昇格
　　↓
タイムラインに「@○○の正体が判明しました」演出が流れる 🎉
```

---

### 3.4 「怪しいやつ」のまま居続けることへの演出

- 長期間未認証のままだと「謎のベテラン」称号がつく（180日以上）
- 認証しないことも一つのキャラクター戦略として成立
- NNCの逆張り文化と相性がよく、あえて認証しない層が生まれる

## 4. 機能定義（How）

### 4.1 機能一覧と優先度

```
【MVP（フェーズ1）】 ─ まず作るべきコア機能
　├─ F-01 会員登録・認証（二層化）
　├─ F-02 サボり投稿 + いいね
　├─ F-03 No-ES Streak（連続サボり記録）
　├─ F-04 チキンレース（脱落管理）
　└─ F-05 汚染度システム + キャラ進化

【フェーズ2】 ─ エンゲージメント向上
　├─ F-06 脱落者の墓標・訃報演出
　├─ F-07 全国・大学別サボりランキング
　├─ F-08 就活警察・密告機能
　└─ F-09 デイリー投稿フォーマット

【フェーズ3（バズ狙い）】 ─ バイラル・話題性
　├─ F-10 サボりガチャ
　├─ F-11 サボり賭け市場
　├─ F-12 脱落者インタビューコーナー
　└─ F-13 親LINEBot
```

---

### 4.2 機能詳細

---

#### F-01｜会員登録・認証（二層化）

**概要：** 大学メールまたは普通メールで登録し、認証ステータスに応じて機能を分ける

| 項目 | 内容 |
|---|---|
| 登録方法A | 大学メール（.ac.jp）認証 → 🎓正規就活生ステータス |
| 登録方法B | 普通メール or Google/Apple SSO → 🕵️怪しいやつステータス |
| プロフィール項目 | ニックネーム、大学名（任意）、学部（任意）、学年（任意） |
| 匿名性 | ニックネームのみ公開、メールアドレスは非公開 |
| アップグレード | 後から大学メールを追加認証してステータス昇格が可能 |
| 退会・出禁 | 内定承諾後は「出禁」処理（アカウント凍結・称号「裏切り者」付与） |

---

#### F-02｜サボり投稿 + いいね（Sabori.log）

**概要：** 就活の代わりに何をしたかを投稿し、共感いいねを競う

| 項目 | 内容 |
|---|---|
| 投稿内容 | テキスト（最大140文字）+ 任意で画像 |
| いいね種類 | 「わかる」「尊い」「草」の3種類のリアクション |
| タイムライン | 全体 / フォロー中 / 大学内 の3種類 |
| 投稿制限 | 🎓：無制限　🕵️：1日3回まで |
| 禁止事項 | 個人特定・誹謗中傷・企業名の直接批判 |

---

#### F-03｜No-ES Streak（連続サボり記録）

**概要：** 何もしない日が続くほどストリークが伸びる、Duolingoの逆

| 項目 | 内容 |
|---|---|
| カウント方法 | 毎日0時にチェックイン（就活行動なし）で+1日 |
| リセット条件 | 汚染ポイント加算行動を申告した時点でリセット |
| 対象 | 🎓：フル機能・称号あり　🕵️：カウントのみ・称号なし |

**称号テーブル（🎓正規就活生のみ）**

| 連続日数 | 称号 | 演出 |
|---|---|---|
| 1〜2日 | 様子見中 | — |
| 3〜6日 | 準備運動中 | — |
| 7日 | 1週間の猛者 | 🎉バッジ付与 |
| 30日 | 悟りの境地 | 🏅バッジ付与・ランキング掲載 |
| 60日 | 無敵モード | 🔥バッジ付与 |
| 100日 | 伝説のサボリスト | 👑称号・殿堂入り |

---

#### F-04｜チキンレース（Last One Standing）

**概要：** 就活を一番最後まで始めなかった人が優勝するゲーム（🎓正規就活生のみ参加可）

| 項目 | 内容 |
|---|---|
| 参加資格 | 🎓正規就活生のみ（🕵️怪しいやつは観戦のみ） |
| 参加方法 | シーズン開始時にエントリー（任意） |
| シーズン期間 | 就活解禁日〜内定式（約8ヶ月）を1シーズンとする |
| 脱落条件 | ES提出・インターン参加などを自己申告した時点で脱落 |
| 脱落演出 | タイムラインに訃報風お知らせが自動投稿される |
| 優勝条件 | 最後まで残った1人（または最長サボり記録者）が優勝 |
| 優勝称号 | 「令和の無職王」（殿堂入り・プロフィールに永久表示） |
| リアルタイム表示 | 現在の生存者数・脱落者数をトップページに常時表示 |

**訃報テンプレート**
```
━━━━━━━━━━━━━━━━━━
　　　　訃　報

　　@{ユーザー名} 氏は
　　本日、{脱落理由} されました。

　　在サボり日数：{日数}日
　　汚染度：{ポイント}pt

　　ご冥福をお祈りいたします。
━━━━━━━━━━━━━━━━━━
```

---

#### F-05｜汚染度システム + キャラ進化

**概要：** 就活行動をするほど「汚染」されてキャラが変化する逆進化システム（🎓正規就活生のみ）

**汚染ポイント定義**

| 就活行動 | 汚染ポイント | 申告方法 |
|---|---|---|
| ES提出（1社） | +10pt | 自己申告 |
| 説明会参加 | +5pt | 自己申告 |
| インターン参加（短期・1日〜2週間） | +20pt | 自己申告 |
| インターン参加（長期・1ヶ月以上） | +50pt | 自己申告 |
| OB/OG訪問 | +15pt | 自己申告 |
| SPIの勉強（1時間以上） | +8pt | 自己申告 |
| リクルートスーツを購入 | +30pt | 自己申告 |
| 内定獲得 | **即時ゲームオーバー** | 自己申告 |

**キャラ進化テーブル**

| 汚染度 | キャラ名 | 説明 | アイコン |
|---|---|---|---|
| 0pt | 純粋な魂 | まだ汚染されていない清廉な存在。みんなから愛される | 🌿 |
| 10〜34pt | スーツの亡霊 | うっすらスーツが透けて見え始める。仲間から「大丈夫？」と心配される | 💼 |
| 35〜99pt | マイナビの奴隷 | 目が死んでいる。口癖が「御社の〇〇という点に魅力を感じ」になっている | 🤖 |
| 100〜199pt | ガクチカゾンビ | ガクチカを語り始めると止まらない。仲間が距離を置き始める | 🧟 |
| 内定獲得 | 社畜の卵（出禁） | アカウント永久凍結。称号「裏切り者」付与 | 👔⛔ |

**タイムライン上の演出**
- 汚染度10pt以上：投稿に「⚠️汚染注意」バッジ表示
- 汚染度35pt以上：投稿に「🚨就活汚染注意🚨」バナー自動付与
- 仲間から「浄化スタンプ」を送れる（「まだ間に合う」「帰ってこい」）

## 5. データモデル（詳細）

```sql
-- ユーザーテーブル
Users
  user_id          UUID PRIMARY KEY
  nickname         VARCHAR(50) NOT NULL
  contact_email    VARCHAR(255) NOT NULL UNIQUE   -- ログイン用メール
  university_email VARCHAR(255) UNIQUE             -- 大学メール（任意）
  auth_type        ENUM('university', 'normal')    -- 認証ステータス
  university       VARCHAR(100)                    -- 大学名（任意・自己申告）
  faculty          VARCHAR(100)                    -- 学部（任意・自己申告）
  grade            SMALLINT                        -- 学年（任意・自己申告）
  contamination_pt INTEGER DEFAULT 0               -- 汚染ポイント
  character_stage  ENUM('pure','ghost','slave','zombie','banned')
  streak_days      INTEGER DEFAULT 0               -- 連続サボり日数
  is_banned        BOOLEAN DEFAULT FALSE           -- 出禁フラグ
  created_at       TIMESTAMP DEFAULT NOW()

-- 投稿テーブル
Posts
  post_id          UUID PRIMARY KEY
  user_id          UUID REFERENCES Users
  content          VARCHAR(140) NOT NULL
  image_url        VARCHAR(500)
  post_type        ENUM('normal', 'daily', 'elimination')
  created_at       TIMESTAMP DEFAULT NOW()

-- リアクションテーブル
Reactions
  reaction_id      UUID PRIMARY KEY
  post_id          UUID REFERENCES Posts
  user_id          UUID REFERENCES Users
  reaction_type    ENUM('wakaru', 'toutoi', 'kusa')
  created_at       TIMESTAMP DEFAULT NOW()

-- チキンレーステーブル
ChickenRace
  entry_id         UUID PRIMARY KEY
  season_id        UUID REFERENCES Seasons
  user_id          UUID REFERENCES Users
  status           ENUM('alive', 'eliminated')
  survived_days    INTEGER DEFAULT 0
  eliminated_at    TIMESTAMP

-- シーズンテーブル
Seasons
  season_id        UUID PRIMARY KEY
  name             VARCHAR(100)

  inform_id        UUID PRIMARY KEY
```

## 8. リスクと対策

| リスク | 対策 |
|---|---|
| 就活時期の集中問題 | インターン活動期（1〜2年生）に対象を広げ通年化 |
| 誹謗中傷・炎上 | 匿名性管理・通報モデレーション機能 |
| 本当に就活しなくなる | 「やはり就活した」報告がコンテンツになる仕組みで救済 |
| 内定後の離脱 | 内定者が「裏切り者」として語り草になるコンテンツ化 |
| 汚染度の虚偽申告 | 自己申告制のため完全防止は困難。コミュニティ文化として許容 |
| 大学メール失効（卒業後） | contact_emailとuniversity_emailを分離管理し、失効しても継続利用可能 |
| 🕵️ユーザーの荒らし | 通常投稿を1日3回に制限・リアクション制限で影響を最小化 |
