# Naitei Nai Club (NNC) — 要求・要件・機能定義書

> *「内定ない」を誇れる唯一のSNS*

---

## 1. 要求定義（Why）

### 1.1 背景・課題

| 課題 | 詳細 |
|---|---|
| 就活の心理的ハードル | 「やらなければ」という圧力が逆に萎縮・無気力を生む |
| 孤独感 | 同じゼミ・学部で志望先が違うと相談しにくい |
| 就活うつ | 精神的なプレッシャーで体調・メンタルを崩す学生が増加 |
| 既存サービスの限界 | 就活媒体は情報閲覧・縦のつながり中心でリアルタイムの横のつながりが弱い |

### 1.2 解決アプローチ

```
「プレッシャーをかける」ではなく「笑いながら就活を始められる」環境を作る

逆インセンティブ設計：
　就活行動 → 嘲笑されるキャラになる
　　↓
　ネタとして就活が語れるようになる
　　↓
　心理的ハードルが下がり、自然と動き始める
```

### 1.3 ビジョン

> 就活をソロゲームからチキンレースに変え、笑いながら乗り越えられるコミュニティを作る

### 1.4 ターゲットユーザー

| 区分 | 詳細 |
|---|---|
| 主ターゲット | 大学3年生・大学院1年生（就活解禁前後） |
| 副ターゲット | 大学1〜2年生（インターン活動期・通年利用） |
| 除外 | 既に内定承諾済みの学生（サービス出禁） |

## 2. 要件定義（What）

### 2.1 ビジネス要件

| 要件ID | 内容 | 優先度 |
|---|---|---|
| BR-01 | 大学メール（.ac.jp）認証による実在確認・信頼性担保 | 高 |
| BR-02 | 就活時期（3〜6月）だけでなく通年利用できる設計 | 中 |
| BR-03 | SNSシェアによるバイラル拡散を前提とした設計 | 高 |
| BR-04 | 誹謗中傷・炎上リスクへの対策 | 高 |

### 2.2 ユーザー要件

| 要件ID | ユーザーストーリー | 優先度 |
|---|---|---|
| UR-01 | 就活生として、今日サボった内容を投稿して共感を得たい | 高 |
| UR-02 | 就活生として、自分の就活しなさっぷりを数値で確認したい | 高 |
| UR-03 | 就活生として、他の人も同じようにサボっていることを見て安心したい | 高 |
| UR-04 | 就活生として、チキンレースで最後まで残って称号を得たい | 高 |
| UR-05 | 就活生として、就活行動によってキャラが変化していく様子を楽しみたい | 中 |
| UR-06 | 就活生として、脱落者（ES提出者）の動向をリアルタイムで知りたい | 中 |
| UR-07 | 就活生として、大学別・学部別のサボりランキングを見たい | 低 |

### 2.3 システム要件

| 要件ID | 内容 | 優先度 |
|---|---|---|
| SR-01 | 大学メール認証による会員登録 | 高 |
| SR-02 | 投稿・いいね・コメントのリアルタイム反映 | 高 |
| SR-03 | 脱落イベント発生時のリアルタイム通知（WebSocket） | 高 |
| SR-04 | 汚染ポイントの自動計算・キャラ状態の自動更新 | 高 |
| SR-05 | 連続サボり日数（Streak）の毎日自動カウント | 高 |
| SR-06 | 大学別・学部別の集計・ランキング生成 | 中 |
| SR-07 | 不適切コンテンツの通報・モデレーション機能 | 高 |
| SR-08 | スマートフォン対応（レスポンシブデザイン） | 高 |

### 2.4 非機能要件

| 区分 | 要件 |
|---|---|
| 可用性 | 99%以上の稼働率（就活解禁期は特に重要） |
| パフォーマンス | 投稿・通知のレスポンスは1秒以内 |
| セキュリティ | 個人情報の匿名化・大学メール以外のログイン禁止 |
| スケーラビリティ | 就活解禁時期のアクセス集中に対応できる設計 |
| プライバシー | 汚染度・サボり記録は本人が公開範囲を選択できる |

## 3. 機能定義（How）

### 3.1 機能一覧と優先度

```
【MVP（フェーズ1）】 ─ まず作るべきコア機能
　├─ F-01 会員登録・認証
　├─ F-02 サボり投稿 + いいね
　├─ F-03 No-ES Streak（連続サボり記録）
　├─ F-04 チキンレース（脱落管理）
　└─ F-05 汚染度システム + キャラ進化

【フェーズ2】 ─ エンゲージメント向上
　├─ F-06 脱落者の墓標・訃報演出
　├─ F-07 全国・大学別サボりランキング
　├─ F-08 就活警察・密告機能
　└─ F-09 デイリー投稿フォーマット

【フェーズ3（バズ狙い）】 ─ バイラル・話題性
　├─ F-10 サボりガチャ
　├─ F-11 サボり賭け市場
　├─ F-12 脱落者インタビューコーナー
　└─ F-13 親LINEBot
```

### 3.2 機能詳細

---

#### F-01｜会員登録・認証

**概要：** 大学メールアドレスで登録し、学生であることを担保する

| 項目 | 内容 |
|---|---|
| 登録方法 | 大学メール（.ac.jp）でのメール認証 |
| プロフィール項目 | ニックネーム、大学名、学部、学年 |
| 匿名性 | ニックネームのみ公開、実名は非公開 |
| 退会 | 内定承諾後は「出禁」処理（アカウント凍結・称号付与） |

---

#### F-02｜サボり投稿 + いいね（Sabori.log）

**概要：** 就活の代わりに何をしたかを投稿し、共感いいねを競う

| 項目 | 内容 |
|---|---|
| 投稿内容 | テキスト（最大140文字）+ 任意で画像 |
| いいね種類 | 「わかる」「尊い」「草」の3種類のリアクション |
| タイムライン | 全体 / フォロー中 / 大学内 の3種類 |
| 投稿制限 | 1日3投稿まで（デイリー投稿の価値を保つため） |
| 禁止事項 | 個人特定・誹謗中傷・企業名の直接批判 |

---

#### F-03｜No-ES Streak（連続サボり記録）

**概要：** 何もしない日が続くほどストリークが伸びる、Duolingoの逆

| 項目 | 内容 |
|---|---|
| カウント方法 | 毎日0時にチェックイン（就活行動なし）で+1日 |
| リセット条件 | 汚染ポイント加算行動を申告した時点でリセット |
| 称号テーブル | 下表参照 |

**称号テーブル**

| 連続日数 | 称号 | 演出 |
|---|---|---|
| 1〜2日 | 様子見中 | — |
| 3〜6日 | 準備運動中 | — |
| 7日 | 1週間の猛者 | 🎉バッジ付与 |
| 30日 | 悟りの境地 | 🏅バッジ付与・ランキング掲載 |
| 60日 | 無敵モード | 🔥バッジ付与 |
| 100日 | 伝説のサボリスト | 👑称号・殿堂入り |

---

#### F-04｜チキンレース（Last One Standing）

**概要：** 就活を一番最後まで始めなかった人が優勝するゲーム

| 項目 | 内容 |
|---|---|
| 参加方法 | シーズン開始時にエントリー（任意） |
| シーズン期間 | 就活解禁日〜内定式（約8ヶ月）を1シーズンとする |
| 脱落条件 | ES提出・インターン参加などを自己申告した時点で脱落 |
| 脱落演出 | タイムラインに訃報風お知らせが自動投稿される |
| 優勝条件 | 最後まで残った1人（または最長サボり記録者）が優勝 |
| 優勝称号 | 「令和の無職王」（殿堂入り・プロフィールに永久表示） |
| リアルタイム表示 | 現在の生存者数・脱落者数をトップページに常時表示 |

**訃報テンプレート**
```
━━━━━━━━━━━━━━━━━━
　　　　訃　報

　　@{ユーザー名} 氏は
　　本日、{脱落理由} されました。

　　在サボり日数：{日数}日
　　汚染度：{ポイント}pt

　　ご冥福をお祈りいたします。
━━━━━━━━━━━━━━━━━━
```

#### F-05｜汚染度システム + キャラ進化

**概要：** 就活行動をするほど「汚染」されてキャラが変化する逆進化システム

**汚染ポイント定義**

| 就活行動 | 汚染ポイント | 申告方法 |
|---|---|---|
| ES提出（1社） | +10pt | 自己申告 |
| 説明会参加 | +5pt | 自己申告 |
| インターン参加（短期・1日〜2週間） | +20pt | 自己申告 |
| インターン参加（長期・1ヶ月以上） | +50pt | 自己申告 |
| OB/OG訪問 | +15pt | 自己申告 |
| SPIの勉強（1時間以上） | +8pt | 自己申告 |
| リクルートスーツを購入 | +30pt | 自己申告 |
| 内定獲得 | **即時ゲームオーバー** | 自己申告 |

**キャラ進化テーブル**

| 汚染度 | キャラ名 | 説明 | アイコン |
|---|---|---|---|
| 0pt | 純粋な魂 | まだ汚染されていない清廉な存在。みんなから愛される | 🌿 |
| 10〜34pt | スーツの亡霊 | うっすらスーツが透けて見え始める。仲間から「大丈夫？」と心配される | 💼 |
| 35〜99pt | マイナビの奴隷 | 目が死んでいる。口癖が「御社の〇〇という点に魅力を感じ」になっている | 🤖 |
| 100〜199pt | ガクチカゾンビ | ガクチカを語り始めると止まらない。仲間が距離を置き始める | 🧟 |
| 内定獲得 | 社畜の卵（出禁） | アカウント永久凍結。称号「裏切り者」付与 | 👔⛔ |

**タイムライン上の演出**
- 汚染度10pt以上：投稿に「⚠️汚染注意」バッジ表示
- 汚染度35pt以上：投稿に「🚨就活汚染注意🚨」バナー自動付与
- 仲間から「浄化スタンプ」を送れる（「まだ間に合う」「帰ってこい」）

#### F-06｜脱落者の墓標・訃報演出

**概要：** チキンレースで脱落した人を「墓標」として記念する

| 項目 | 内容 |
|---|---|
| 墓標表示 | プロフィールに👼マーク・「○○日目に散る」が永久表示 |
| 追悼コメント | 脱落投稿にコメントが集まる（「ご冥福をお祈りします」等） |
| 墓地ページ | 全脱落者の墓標一覧ページ（名誉の殿堂の逆バージョン） |

---

#### F-07｜全国・大学別サボりランキング

**概要：** 大学・学部・地域別のサボり偏差値を集計・公開する

| 項目 | 内容 |
|---|---|
| 集計軸 | 大学別 / 学部別 / 地域別 / 学年別 |
| 指標 | 平均サボり日数・平均汚染度・チキンレース生存率 |
| 更新頻度 | 毎週月曜日に週次ランキング更新 |
| プライバシー | 個人は特定できない集計データのみ公開 |

---

#### F-08｜就活警察・密告機能

**概要：** 仲間の「就活の気配」を密告してコミュニティで笑える機能

| 項目 | 内容 |
|---|---|
| 密告内容 | 「スーツ姿を見た」「リクナビ開いてた」等のテンプレ選択式 |
| 密告先 | 特定ユーザーへの匿名密告 |
| 被密告時の演出 | 「就活警察から通報が届きました」という警告通知 |
| 制裁ポイント | 密告が3件集まると汚染ポイント+5（疑惑ポイント） |

---

#### F-09｜デイリー投稿フォーマット

**概要：** 毎日1回だけ投稿できる「今日のサボり報告」

| 項目 | 内容 |
|---|---|
| フォーマット | スキップした就活行動 / 代わりにやったこと / 一言コメント |
| 投稿制限 | 1日1回のみ（Wordle的な希少性） |
| 集計 | デイリー人気投稿をトップページに掲載 |

例：
```
📋 今日のサボり報告
　スキップ：説明会3社・SPI練習
　代わりに：Netflix4時間・ラーメン二郎
　一言：後悔はない
```

---

#### F-10｜サボりガチャ

**概要：** 毎日ログインするとランダムで「今日のサボり免罪符」が出る

| 項目 | 内容 |
|---|---|
| 実行タイミング | 毎日初回ログイン時に自動実行 |
| 結果例 | 「本日はOB訪問をスキップしても運勢は大吉です」 |
| レアリティ | 大吉〜凶の6段階（「就活解禁」は最凶） |
| シェア機能 | 結果画像をSNSにシェアできる（バイラル設計） |

---

#### F-11｜サボり賭け市場

**概要：** 「このユーザーはあと何日で脱落するか」をポイントで予測する

| 項目 | 内容 |
|---|---|
| 対象 | チキンレース参加中の全ユーザー |
| 予測方法 | 「○日以内に脱落する/しない」をポイントで賭ける |
| 精算 | 予測が当たればポイント還元（賭博にならないよう仮想ポイントのみ） |
| 表示 | 各ユーザーの「脱落確率オッズ」が表示される |

---

#### F-12｜脱落者インタビューコーナー

**概要：** チキンレースで脱落した人が「なぜ就活を始めたか」を語る読み物コンテンツ

| 項目 | 内容 |
|---|---|
| 形式 | テキストインタビュー（Q&A形式） |
| 主な質問 | 何がきっかけで就活を始めたか / 後悔はあるか / 仲間へのメッセージ |
| 掲載 | 週1本、編集部（運営）がピックアップして掲載 |
| 目的 | 読み物コンテンツとしてSNSでバズる・新規流入を促す |

---

#### F-13｜親LINEBot

**概要：** 「就職どうなの？」という圧力LINEをランダムで送ってくるBot

| 項目 | 内容 |
|---|---|
| 配信頻度 | ランダム（週1〜3回） |
| メッセージ例 | 「就職活動はどう？」「同級生の○○くんはもう内定もらったって」 |
| ユーザーアクション | 受け取ったBotのメッセージをスクショ投稿してネタにできる |
| ON/OFF | ユーザーが任意でON/OFF切替可能 |

## 4. データモデル（概略）

```
Users
　├─ user_id
　├─ email（大学メール）
　├─ nickname
　├─ university / faculty / grade
　├─ contamination_point（汚染度）
│  ├─ streak_days（連続サボり日数）
　├─ character_stage（キャラステージ）
　├─ is_banned（出禁フラグ）
　└─ created_at

Posts
　├─ post_id
　├─ user_id
　├─ content
　├─ post_type（通常 / デイリー / 脱落報告）
　└─ created_at

Reactions
　├─ reaction_id
　├─ post_id
　├─ user_id
　└─ reaction_type（わかる / 尊い / 草）

ChickenRace
　├─ season_id
　├─ user_id
　├─ status（生存 / 脱落）
　├─ survived_days
　└─ eliminated_at

ContaminationLogs
　├─ log_id
　├─ user_id
　├─ action_type（ES提出 / 説明会 / etc.）
　├─ point_added
　└─ created_at
```

## 6. リスクと対策

| リスク | 対策 |
|---|---|
| 就活時期の集中問題 | インターン活動期（1〜2年生）に対象を広げ通年化 |
| 誹謗中傷・炎上 | 匿名性管理・大学メール認証・通報モデレーション機能 |
| 本当に就活しなくなる | 「やはり就活した」報告がコンテンツになる仕組みで救済 |
| 内定後の離脱 | 内定者が「裏切り者」として語り草になるコンテンツ化 |
| 汚染度の虚偽申告 | 自己申告制のため完全防止は困難。コミュニティ文化として許容 |
